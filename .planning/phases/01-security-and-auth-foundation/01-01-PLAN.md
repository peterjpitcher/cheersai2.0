    - Test 7: getKey() returns a 32-byte Buffer from a hex-encoded env var
    - Test 8: getKey() throws if TOKEN_VAULT_KEY is not set or not 64 hex chars
  </behavior>
  <action>
    Create standalone token vault module at `src/lib/token-vault/` per D-01 and D-02.

    **src/lib/token-vault/types.ts:**
    - Export `interface EncryptedPayload { ciphertext: string; iv: string; tag: string; keyVersion: number; }`
    - Export `interface VaultConfig { keyEnvVar: string; }`
    - All string fields are base64-encoded

    **src/lib/token-vault/crypto.ts:**
    - `import crypto from 'node:crypto';`
    - Constants: `ALGORITHM = 'aes-256-gcm'`, `IV_LENGTH = 12`, `TAG_LENGTH = 16`
    - `export function encryptPayload(plaintext: string, key: Buffer, keyVersion: number): EncryptedPayload` -- generate random 12-byte IV via `crypto.randomBytes(IV_LENGTH)`, create cipher with `crypto.createCipheriv(ALGORITHM, key, iv)`, concatenate update+final, extract auth tag, return base64-encoded fields with keyVersion
    - `export function decryptPayload(payload: EncryptedPayload, key: Buffer): string` -- decode base64 IV/ciphertext/tag, create decipher, call `setAuthTag()` BEFORE update (per Pitfall 6 in RESEARCH.md), return UTF-8 string

    **src/lib/token-vault/key-management.ts:**
    - `export function getKey(envVar: string = 'TOKEN_VAULT_KEY'): Buffer` -- read from `process.env[envVar]`, validate it is exactly 64 hex chars, return `Buffer.from(hexString, 'hex')` (32 bytes for AES-256)
    - `export function getCurrentKeyVersion(): number` -- read `process.env.TOKEN_VAULT_KEY_VERSION`, parse as integer, default to 1 if not set
    - Support future multi-key rotation: key env var naming convention is `TOKEN_VAULT_KEY` (current) and `TOKEN_VAULT_KEY_V{N}` (old versions for lazy re-encrypt)

    **src/lib/token-vault/index.ts:**
    - Public API re-exports: `encrypt(plaintext: string): EncryptedPayload`, `decrypt(payload: EncryptedPayload): string`, `rotate(payload: EncryptedPayload, oldKey: Buffer, oldKeyVersion: number): EncryptedPayload`
    - `encrypt()` calls `getKey()` and `getCurrentKeyVersion()` internally
    - `decrypt()` calls `getKey()` -- if `payload.keyVersion` does not match current version, caller should call `rotate()` (lazy re-encrypt strategy per RESEARCH.md Discretion recommendation)
    - `rotate()` decrypts with oldKey, encrypts with current key, returns new payload

    **src/lib/token-vault/crypto.test.ts:**
    - Use `describe('token-vault')` with nested describes per function
    - Generate test key: `crypto.randomBytes(32)` -- do NOT use process.env in tests
    - Pass key directly to `encryptPayload`/`decryptPayload` to avoid env coupling
    - All 8 behavior tests listed above
    - Use `vi.stubEnv()` for key-management tests
