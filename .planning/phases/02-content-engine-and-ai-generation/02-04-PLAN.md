**2. Create `src/features/create/create-wizard.tsx`:**
- `'use client'` component
- Props: `{ initialDraftId?: string; onClose: () => void }`
- State: `currentStep` (0-3), `contentType` (null initially), `draftId` (null or string), `formData` (accumulated brief data)
- Uses React Hook Form with Zod resolver for the brief step
- Uses `useAutoSaveDraft(draftId)` to auto-save on step transitions (D-03)
- Step labels: ['Brief', 'Generate', 'Media', 'Schedule']
- Step indicator: horizontal progress bar showing current step (1-4) with labels
- Navigation: "Back" button (goes to previous step), "Next" / "Continue" button (validates current step, advances)
- On step 0 (Brief): renders type picker + type-specific fields
- On step 1 (Generate): renders GenerateStep placeholder (will be wired in Plan 05)
- On step 2 (Media): renders MediaStep placeholder (will be wired in Plan 06)
- On step 3 (Schedule): renders ScheduleStep
- On first "Next" from step 0: calls `createDraft(formData)` to get a `draftId`, then saves and advances
- On subsequent step changes: calls `save(draftState)` from auto-save hook
- If `initialDraftId` provided: load draft via `getDraft(initialDraftId)` and resume at saved step
- Framer Motion `AnimatePresence` for step transitions (D-10): slide-left when advancing, slide-right when going back
