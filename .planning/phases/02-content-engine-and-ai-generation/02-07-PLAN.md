Rebuild conflict detection from v1 reference patterns. Clean implementation, not copy.

```typescript
import { DateTime } from 'luxon';
import { DEFAULT_TIMEZONE } from '@/lib/constants';
import type { ContentItem, Platform } from '@/types/content';

export interface Conflict {
  itemA: { id: string; title: string | null; scheduledAt: Date };
  itemB: { id: string; title: string | null; scheduledAt: Date };
  platform: Platform;
  gapMinutes: number;
  suggestion: string; // "Move 'Item B' to 14:30" or similar
}

const CONFLICT_WINDOW_MINUTES = 30;

export function detectConflicts(items: ContentItem[]): Conflict[]
```
- Filter to items with `scheduledAt` not null
- Group by platform (each item may target multiple platforms via body_draft.platforms)
- For each platform group: sort by scheduledAt ascending
- Compare adjacent pairs: if gap < 30 minutes, create Conflict entry
- Suggestion: "Move '{itemB.title}' to {itemA.scheduledAt + 30 min}" formatted in Europe/London timezone using Luxon
- All time comparisons must use `DateTime.fromJSDate(date, { zone: DEFAULT_TIMEZONE })` (SCHED-05)

**2. Create `src/lib/scheduling/materialise.ts`** (SCHED-03):

Expand recurring campaigns into individual calendar slots.

```typescript
import { DateTime } from 'luxon';
import { DEFAULT_TIMEZONE } from '@/lib/constants';
import type { ContentItem } from '@/types/content';

export interface MaterialisedSlot {
  sourceId: string;        // ID of the recurring campaign
  scheduledAt: Date;       // The specific materialised date/time
  contentType: 'weekly_recurring';
  title: string | null;
  status: ContentItem['status'];
}

export function materialiseRecurring(
  recurring: ContentItem[],
  rangeStart: DateTime,
  rangeEnd: DateTime,
): MaterialisedSlot[]
```
- For each recurring item: read `recurringDayOfWeek` (0=Sunday, 6=Saturday) and time from body_draft
- Starting from rangeStart, iterate week by week up to rangeEnd
- For each week: find the date matching dayOfWeek, set time in Europe/London timezone
- Generate a MaterialisedSlot for each
- Use `weeksAhead` from body_draft to limit how far ahead to materialise (default 4 weeks)
- Respect the campaign's status: if 'draft', materialised slots show as 'draft'

All Luxon operations use `{ zone: DEFAULT_TIMEZONE }` (SCHED-05).
