## Auth Model Summary

- **Session auth**: Supabase JWT stored in HTTP-only cookies. All protected routes call `getSupabaseServerClient()` + `auth.getUser()` server-side.
- **Cron auth**: All cron API routes validate `Authorization: Bearer <CRON_SECRET>` header before executing.
- **OAuth flows**: Social provider connections (Facebook, Instagram, Google) use state stored in `oauth_states` table; callbacks handled at `/api/oauth/[provider]/callback` and `/api/oauth/facebook-ads/callback`.
- **Middleware** (`src/middleware.ts`): Handles only apex-to-www redirect (`cheersai.uk` → `www.cheersai.uk`). No auth enforcement at middleware layer — auth is checked inside layout files and server actions.
