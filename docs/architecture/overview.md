## Auth Model (summary)

- **Edge middleware** (`middleware.ts`): only redirects `cheersai.uk` apex to `www.cheersai.uk`. **It does not perform auth.**
- **Layout-level auth** is the real gate:
  - `src/app/(app)/layout.tsx` calls `getCurrentUser()` from `src/lib/auth/server.ts` for every authenticated page; redirects to `/login` if no user.
  - `src/app/(auth)/layout.tsx` redirects already-signed-in users away from `/login`.
  - `src/app/(public)/layout.tsx` and `src/app/layout.tsx` are unauthenticated.
- **Server actions and API routes** re-verify auth via `requireAuthContext()` before any mutation.
- **Cron auth**: 7 cron routes plus `api/internal/render-banner` validate `process.env.CRON_SECRET` against an `Authorization` header (no user session).
- **OAuth callbacks** (`/api/oauth/[provider]/callback`, `/api/oauth/facebook-ads/callback`) verify a signed `state` cookie and exchange the code via the Supabase service-role client.

See [[relationships]] for the integration → file map and [[server-actions]] for per-action auth checks.
