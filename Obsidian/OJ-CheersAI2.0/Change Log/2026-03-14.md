# Change Log — 2026-03-14

### 12:00 — Four Tier 2 optimisations implemented

- **What changed**: Second parallel agent team implemented four improvements
- **Files created**:
  - `src/app/api/create/generate-stream/route.ts` (new) — SSE streaming Route Handler
  - `src/features/create/streaming-preview.tsx` (new) — per-platform live text display
  - `src/app/(app)/create/template-actions.ts` (new) — `listTemplates`, `saveTemplate`, `deleteTemplate`, `incrementTemplateUseCount` server actions
  - `src/features/create/template-selector.tsx` (new) — Load/Save saved prompt UI component
  - `src/app/api/cron/notify-expiring-connections/route.ts` (new) — token expiry email cron
  - `supabase/migrations/20260314000002_content_templates.sql` (new) — `content_templates` table + RLS
- **Files modified**:
  - `src/features/create/instant-post-form.tsx` — streaming fetch + TemplateSelector wiring
  - `src/lib/planner/data.ts` — `unstable_cache` for signed URL batch generation
- **Docs updated**: [[Route Handlers]], [[Server Actions]], [[Schema]], [[Optimization Opportunities]]
- **Details**:
  - **Streaming AI generation** — fake timer progress bar replaced with real OpenAI streaming via SSE. `POST /api/create/generate-stream` streams `platform_start → chunk* → platform_done → done` events. Users see copy appearing word-by-word. Double-call design: streams preview then calls `createInstantPost()` for the real save.
  - **Content templates** — pub owners can save prompt templates and load them in one click. `content_templates` table with RLS; `TemplateSelector` component appears below the prompt textarea in the Instant Post basics stage.
  - **Expiring connection emails** — `POST /api/cron/notify-expiring-connections` polls for connections with `expires_at < now() + 7 days`, checks `emailTokenExpiring` preference, idempotent via `notifications` table (7-day dedup window).
  - **Signed URL caching** — `loadPrimaryMediaPreviewsByContent` and `loadMediaPreviews` now use `unstable_cache` with 480s TTL; service-role client used inside cached function to avoid request-context serialisation issues. >99% reduction in Storage API calls per planner session.

### 10:30 — Three Tier 1 optimisations implemented

- **What changed**: Parallel agent team implemented three improvements from the prioritised optimisation list
- **Files affected**:
  - `supabase/migrations/20260314000001_add_notifications_composite_index.sql` (new)
  - `src/lib/email/resend.ts` (new)
  - `src/app/api/cron/notify-failures/route.ts` (new)
  - `src/app/(app)/reviews/actions.ts` (modified)
- **Docs updated**: [[Optimization Opportunities]], [[Tech Debt]], [[Route Handlers]], [[Reviews]]
- **Details**:
  - **Notifications DB index** — Added `notifications_account_unread_idx` partial composite index `(account_id, created_at DESC) WHERE read_at IS NULL`, eliminating full-table scans on every planner load
  - **AI review reply — dynamic brand voice** — Removed hardcoded "The Anchor" system prompt from `generateAiDraft()`; replaced with `buildSystemPrompt()` that reads `getOwnerSettings()` (venue name, tone sliders, banned phrases, key phrases) with graceful fallback on error
  - **Publish failure emails** — Created `sendEmail()` utility (Resend SDK) and `POST /api/cron/notify-failures` cron endpoint; respects per-account email preferences and uses `notifications` table for idempotency to prevent duplicate sends
